こんにちは、ナウキャストでリードエンジニアをしている六車です。

ナウキャストでは、エンジニア採用の選考ステップの一つとして、書類選考を通過したエンジニア候補の方にスキルテストを受けていただいています。

この度、そのスキルテストをリニューアルしました。具体的には、グループ会社であるFinatextが運用している**バックエンドエンジニア向けスキルテスト**（[参考記事](https://techblog.finatext.com/new-skill-test-e7a0f0247a84)）をベースに、ナウキャストの**データエンジニアおよびLLMエンジニア**の採用にも導入することにしました。

「データエンジニアやLLMエンジニアなのに、なんでバックエンドエンジニア向けのテストを？」と思われた方もいるかもしれません。この記事では、そのテスト内容と、なぜそれを両職種の採用に取り入れたのか、その意図をお伝えします。

---

## Finatextのスキルテストとは

Finatextのスキルテストは、CTF（Capture The Flag）形式で出題される実践的な内容です。概要は以下の通りです。

- **制限時間：72時間**（本人の都合に合わせて進められる）
- **形式：** 問題を解くとフラグが出てくる CTF 形式
- **題材：** コンテナ（Docker）、JWT 認証など、実務で使われるモダンな技術
- **サポート：** 質問はベストエフォートで対応（アテンド担当なし）
- **テスト後：** レビュー会でどのように解いたかを説明してもらう

評価の観点は以下です。

1. 技術力があるか、または知らなくても**調べて解決できる能力があるか**
2. 詰まったときに、適切に助けを求めて前に進めるか
3. 解いた方法や詰まった箇所を、**わかりやすく人に説明できるか**
4. 「この人となら一緒に仕事したい」と思えるか

詳細はFinatextのテックブログをご覧ください。

## なぜデータエンジニア・LLMエンジニアがバックエンド向けのテストを？

正直に言うと、このテストはデータエンジニアやLLMエンジニアが日常的に扱う仕事内容とは直接は関係しません。dbt や Snowflake の問題でもなく、RAG のチューニングやプロンプト設計でもない。

それでもこのテストを導入したのには、理由があります。

### AI駆動開発の時代こそ、ソフトウェアエンジニアリングの基礎が問われる

生成AIを使った開発が当たり前になってきた今、ツールの使い方を覚えるだけでスペシャリストとしてやっていける時代は終わりつつあります。

データエンジニアであっても、コンテナの仕組みを理解していなければAIが生成した Dockerfile を正しくレビューできません。JWT 認証を知らなければ API との連携でハマり続けます。LLMエンジニアについても同様で、LLM を呼び出すだけのコードを書けても、それを本番で動かすための設計力・実装力がなければプロダクトには繋がりません。プロンプトエンジニアリングとソフトウェアエンジニアリングは、本来切り離せないものです。

ナウキャストでは、データエンジニアもLLMエンジニアも、クライアント向けのソリューション開発に直接関わります。バックエンドエンジニアやインフラエンジニアと協働する場面も多く、**ソフトウェアエンジニアとしての地力**は両職種に共通して求められる素養です。

**職種の看板がデータであれLLMであれ、ソフトウェアエンジニアとしての基礎を大切にしたい。** それがこのテストを導入した一番の理由です。

### 「知らないこと」への向き合い方を見たい

もう一つの理由は、知らない領域への姿勢です。

このテストの題材は、データエンジニアにとっても、LLMエンジニアにとっても、なじみが薄い部分があるかもしれません。でも、**調べれば解ける**ように設計されています。

私たちが見たいのは、正答率そのものではありません。

- 知らない技術に直面したとき、どうやって調べるか
- 生成AIを使いながら理解を深め、自分の言葉で説明できるか
- 詰まっても諦めずに前に進めるか

今の時代、生成AIを使って調べることは当然のスキルです。ただ、使って「答えを出した」だけでは不十分で、**その答えを理解して、人に伝えられるか**がより重要です。テスト後のレビュー会では、まさにそこを見ます。

### 採用は双方向のプロセス

Finatextのブログにも書かれていましたが、私たちも同じ考え方を持っています。

選考プロセスは、私たちが候補者を評価する場であると同時に、**候補者がナウキャストという会社・チームを見極める場**でもあります。

スキルテストを通じて、どんな技術を大切にしているか、どんな人と一緒に働くことになるのか、そういったことを感じ取っていただければ嬉しいです。

スキルテストを受けていただく皆さんへ。

知らない問題が出ても怖くありません。調べてください。生成AIを使ってください。そして解いた後、**自分が何をやったのかを説明できる状態**にしてください。そこに、私たちが一緒に働きたいと思う人の姿があります。
